# One Day Match  
今日は「誰かと話してもいい日」だけ

## 概要
One Day Match は、  
常時ログインや即時対応を前提としない、  
「その日限りの関わり」を想定したマッチングアプリのプロトタイプです。

毎日使い続けることを目的とせず、  
「今日は少し人と関われそうだ」と感じた日だけ利用できる設計にしています。

---

## 解決したい課題
多くのマッチング・コミュニケーションサービスでは、

- 継続的なログイン
- すぐに返信することへの暗黙の期待
- 通知による心理的な負荷

が前提になりがちです。

その結果、  
人とつながりたい気持ちはあっても、  
やり取りそのものが負担になり、サービスから距離を置いてしまうケースがあると感じました。

---

## コンセプト
One Day Match では、以下を前提に設計しています。

- 常時接続を前提にしない  
- 関係性をその日単位で区切る  
- 使わない日は、存在を意識しなくてよい  

「続けられること」よりも、  
「無理なく使えること」を優先しています。

---

## 主な機能
- ユーザー登録・ログイン
- 最小限のプロフィール作成
- 「今日はOK」ステータスの切り替え
- 当日参加可能なユーザー・投稿の表示
- マッチ・投稿の一覧表示

投稿やマッチには期限の概念を持たせており、  
常に「今日に関係する情報」だけが見える状態を目指しています。  
（実装途中で、表示方法や扱いを調整する可能性があります）

リアルタイムチャットや通知機能は、現時点では実装していません。

---

## 技術スタック
- フレームワーク：Ruby on Rails
- フロントエンド：HTML / CSS / JavaScript
- データベース：SQLite（開発環境）

学習用途・プロトタイプとして、  
構成が理解しやすく、修正しやすい技術を選択しています。

---

## 設計の考え方
本アプリでは、

- 機能を増やすこと
- 滞在時間を伸ばすこと

を目的にしていません。

制約をあらかじめ設けることで、  
ユーザーが「どう使えばいいか」を考えなくて済む状態を作ることを意識しています。

---

## 今後の調整・拡張の余地
- トーンや温度感を選べるステータス
- 投稿内容のテンプレート化
- 利用頻度を自然に抑える仕組み

実装状況や学習進度に応じて、  
機能や仕様を見直しながら進める予定です。

---

## 制作背景
公共分野での業務経験を通じて、  
人間関係の密度や心理的負荷が、行動や選択に大きく影響する場面を多く見てきました。

One Day Match は、  
そうした個人の実感を、  
Webアプリの設計に落とし込むことを目的とした制作物です。

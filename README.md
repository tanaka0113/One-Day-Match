# One Day Match  
今日は「誰かと話してもいい日」だけ

## 🧠 概要（Overview）
One Day Match は、  
**「常時接続・即レスを前提としない」** ことを思想にした、  
低負荷なマッチングアプリのプロトタイプです。

毎日ログインする必要はありません。  
「今日はOK」と感じた日だけ、そっと人とつながれます。

---

## 🎯 解決したい課題（Problem）
多くのマッチング・コミュニケーションサービスは、

- 常時ログイン前提
- 即レス圧力
- 通知による心理的負荷

を暗黙に要求しています。

その結果、

> 人とつながりたいが、  
> 常に誰かとやり取りするのは負担になる

という層が、サービスから静かに離脱していく課題があります。

---

## 💡 コンセプト（Concept）

### ✔ 常時接続を前提にしない
- 毎日ログイン不要  
- 「今日はOK」をONにしたユーザーのみ表示  
- マッチは **24時間で自動消滅**

### ✔ 関係性を“軽く”設計する
- 深いプロフィール不要  
- 会話を急がせない  
- つながりを「その日限り」に限定

---

## 🔑 主な機能（Features）
- ユーザー登録 / ログイン  
- プロフィール作成（最小限）  
- 「今日はOK」ステータス切替  
- 条件マッチング  
- マッチ一覧表示  
- 24時間でマッチ自動削除  

※ リアルタイムチャットは実装していません  
（心理的・技術的な複雑さを意図的に排除）

- 離脱率を下げるための“引き算の設計”

# OneDayMatch

## アプリ概要

OneDayMatchは、当日のスポーツマッチやイベント参加者をマッチングするWebアプリケーションです。

**解決したい課題**：気軽にスポーツ仲間を見つけたいが、数日前から予定を立てるのは負担になる。当日参加可能なメンバーを効率的に見つけたい。

**アプリの特徴**：ユーザーは「今日マッチ可能」な状態を設定でき、一日限りで有効な募集投稿を作成・参加できます。期限が過ぎた投稿は自動的に非表示になるため、常に最新の情報のみが表示されます。

### 主な機能

- **ユーザー認証・登録**: ログイン・ログアウト機能を実装（Deviseという認証ライブラリを使用）
- **当日マッチング**: 「今日参加可能」フラグにより、参加意思を一目で確認できる機能
- **投稿機能**: 一日限りの有効期限を持つ投稿を作成。期限切れの投稿は自動的に非表示
- **いいね機能**: 投稿への関心を示す機能（SNSのいいね機能と同じ）
- **コメント機能**: 投稿者とのコミュニケーションを取る機能

## 設計思想

### 1. 時間制約の明確な管理

一日限りのマッチングを実現するため、以下の設計方針を採用しました：

- **有効期限の自動管理**: 各投稿に「いつまで有効か」を設定し、期限が過ぎた投稿は自動的に非表示にすることで、常に最新情報のみを表示
- **参加意思の明確化**: ユーザーが「今日参加可能」という状態を設定できる機能を追加し、確実なマッチングを実現

この設計により、ユーザーは「今日すぐに」参加できるメンバーを効率的に見つけることができます。

### 2. シンプルで拡張しやすいデータ構造

データベース（データを保存する場所）を必要最小限の4つのテーブルで構成し、将来的な機能追加にも対応しやすい設計にしました：

- **users（ユーザー情報）**: アカウントの基本情報と「今日参加可能」の設定を管理
- **posts（投稿情報）**: 募集内容と有効期限を管理
- **likes（いいね情報）**: 誰がどの投稿にいいねしたかを管理
- **comments（コメント情報）**: コメント内容と投稿・ユーザーの紐付けを管理

各テーブルは「一つの役割だけを担う」という原則で設計しており、機能追加や修正がしやすい構造になっています。

### 3. ユーザー体験の最適化

- 有効期限の自動管理により、古い投稿に惑わされない分かりやすい画面を実現
- コメント機能により、詳細な調整や質問が可能
- いいね機能により、気軽に関心を示すことができる

### 主要テーブル構成

- **users（ユーザー情報テーブル）**: アカウントの基本情報と参加意思を管理
  - `today_ok`: 今日参加可能かどうかの設定（ON/OFF）
  - `display_name`: 表示名
  - `email`, `password_digest`: メールアドレスとパスワード（暗号化済み）

- **posts（投稿テーブル）**: 募集投稿の情報を管理
  - `expires_at`: 投稿の有効期限（一日限りの制約を実現）
  - `content`: 投稿内容（本文）
  - `user_id`: 投稿者を識別するID（どのユーザーが投稿したかを紐付け）

- **likes（いいねテーブル）**: いいね機能のデータを管理
  - `user_id`, `post_id`: どのユーザーがどの投稿にいいねしたかを紐付け

- **comments（コメントテーブル）**: コメント機能のデータを管理
  - `content`: コメント内容
  - `user_id`, `post_id`: どのユーザーがどの投稿にコメントしたかを紐付け

## 技術スタック

このアプリケーションを開発する際に使用した技術やツールです：

- **フレームワーク**: Ruby on Rails 7.1（Webアプリケーションを効率的に開発するためのフレームワーク）
- **認証**: Devise（ログイン・ログアウト機能を簡単に実装できるライブラリ）
- **データベース**: SQLite3（開発環境用の軽量なデータベース）
- **その他**: Bootstrap（見た目を整えるためのUIフレームワーク）

## 開発の意図

本プロダクトは「今日は関われる日だけ関われる」という前提をシステム側で保証することで、ユーザーの心理的負荷を下げることを目的としています。

従来のマッチングアプリでは、数日前から予定を調整する必要がありましたが、このアプリでは「今日参加可能な人だけが表示される」という仕組みにより、気軽に参加できる環境を実現しています。

# One Day Match  
今日は「誰かと話してもいい日」だけ

## 🧠 概要（Overview）
One Day Match は、  
**「常時接続・即レスを前提としない」** ことを思想にした、  
低負荷なマッチングアプリのプロトタイプです。

毎日ログインする必要はありません。  
「今日はOK」と感じた日だけ、そっと人とつながれます。

---

## 🎯 解決したい課題（Problem）
多くのマッチング・コミュニケーションサービスは、

- 常時ログイン前提
- 即レス圧力
- 通知による心理的負荷

を暗黙に要求しています。

その結果、

> 人とつながりたいが、  
> 常に誰かとやり取りするのは負担になる

という層が、サービスから静かに離脱していく課題があります。

---

## 💡 コンセプト（Concept）

### ✔ 常時接続を前提にしない
- 毎日ログイン不要  
- 「今日はOK」をONにしたユーザーのみ表示  
- マッチは **24時間で自動消滅**

### ✔ 関係性を“軽く”設計する
- 深いプロフィール不要  
- 会話を急がせない  
- つながりを「その日限り」に限定

---

## 🔑 主な機能（Features）
- ユーザー登録 / ログイン  
- プロフィール作成（最小限）  
- 「今日はOK」ステータス切替  
- 条件マッチング  
- マッチ一覧表示  
- 24時間でマッチ自動削除  

※ リアルタイムチャットは実装していません  
（心理的・技術的な複雑さを意図的に排除）

## 🧩 技術的なポイント（Technical Highlights）

### 1. 時間制御ロジック
- マッチ生成時に `expired_at` を付与  
- バッチ処理で期限切れデータを自動削除  
- 「永続的な関係」を前提としないDB設計

### 2. 状態ベースのマッチング
- ユーザーの「今日はOK」状態を条件に検索  
- 常時アクティブユーザー前提の設計を回避

### 3. 意図的なUX制限
- 通知最小化  
- 即時コミュニケーションを強制しない  
- 離脱率を下げるための“引き算の設計”

---

## 🧭 設計思想（Design Philosophy）
本アプリは、

> 人とつながりたいが、  
> 常時コミュニケーションは負荷になる

というユーザー課題を想定して設計しています。

**エンゲージメント最大化ではなく、  
ユーザーの心理的安全性を守ること** を優先しています。

---

## 🏗 なぜこの構成にしたか（Why This Design）
- WebSocketやリアルタイム通信を使わず  
  → 保守性・理解しやすさを重視  
- フル機能を目指さず  
  → 思想が伝わる最小構成  
- 制約を「欠点」ではなく  
  → プロダクトの価値として定義
